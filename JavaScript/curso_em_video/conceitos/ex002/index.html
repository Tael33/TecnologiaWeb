1. O que é JavaScript?

JavaScript (JS) é uma linguagem de programação de alto nível, interpretada e dinâmica. É uma das três principais tecnologias da World Wide Web:

  • HTML: Fornece a estrutura (o "esqueleto").
  • CSS: Cuida da apresentação e estilo (a "pele" e "roupas").
  • JavaScript: Adiciona comportamento e interatividade (os "músculos" e "cérebro").

Originalmente criado para rodar apenas nos navegadores (client-side), o JavaScript hoje também é usado no lado do servidor (server-side) graças a ambientes como o Node.js.

2. Como Adicionar JS a uma Página

2.1. Script Externo (Recomendado)

Crie um arquivo .js e o vincule ao HTML. Isso mantém o código organizado e permite cache.

<!-- Colocado geralmente antes de fechar a tag </body> -->
<script src="meu-script.js"></script>

2.2. Script Interno

Código diretamente no HTML dentro da tag <script>.

<script>
    console.log("Olá, mundo!"); // Exibe no console do navegador
</script>

2.3. Inline (Não recomendado)

Código em um atributo de um elemento HTML.

<button onclick="alert('Você clicou!')">Clique em mim</button>

3. Fundamentos da Linguagem

3.1. Variáveis (var, let, const)

Variáveis são "contêineres" para armazenar valores.

  • let: Para valores que podem mudar. Tem escopo de bloco ({}).
  • const: Para valores que não podem mudar (constantes). Tem escopo de bloco.
  • var: A forma antiga. Tem escopo de função. Evite seu uso em código moderno.
let idade = 30;
idade = 31; // Válido

const PI = 3.14159;
// PI = 3.14; // Inválido! Gerará um erro.

3.2. Tipos de Dados

O JavaScript tem tipos de dados primitivos e o tipo objeto.

Primitivos

  • String: Texto. Ex: "Olá", 'Mundo'.
  • Number: Números. Ex: 10, 2.5.
  • Boolean: Valor lógico. Ex: true, false.
  • null: Representa a ausência intencional de um valor. É um valor "nulo" definido pelo programador.
  • undefined: Uma variável que foi declarada mas ainda não teve um valor atribuído.

Não-Primitivo

  • Object: Coleções complexas de dados. Arrays, Funções e Objetos literais são todos do tipo Objeto.

3.3. Operadores

  • Aritméticos: +, -, *, /, % (módulo/resto).
  • Atribuição: =, +=, -=.
  • Comparação (Importante):
    • ==: Igualdade "solta" (compara valor, mas faz coerção de tipo). Ex: 5 == "5" é true. (Evite!)
    • ===: Igualdade "estrita" (compara valor E tipo). Ex: 5 === "5" é false. (Use este!)
    • !=: Diferente.
    • !==: Estritamente diferente. (Use este!)
    • >, <, >=, <=.
  • Lógicos: && (E), || (OU), ! (NÃO).

3.4. Escopo (Scope) e Hoisting

Escopo

Escopo define onde as variáveis e funções são acessíveis no seu código.

  • Escopo Global: Declarado fora de qualquer função. Acessível de qualquer lugar.
  • Escopo de Função: Variáveis declaradas com var (ou funções) são acessíveis apenas dentro da função onde foram criadas.
  • Escopo de Bloco: Variáveis declaradas com let e const são acessíveis apenas dentro do bloco ({}) onde foram criadas (ex: dentro de um if ou for).
let globalVar = "Sou global";

function testeEscopo() {
    var varAntiga = "Escopo de Função";
    let letNova = "Escopo de Bloco";

    if (true) {
        console.log(letNova); // Acessível
        let dentroDoIf = "Também de bloco";
    }
    // console.log(dentroDoIf); // Erro! Fora do escopo.
}
// console.log(varAntiga); // Erro! Fora do escopo.

Hoisting (Içamento)

Hoisting é o comportamento do JavaScript de mover declarações de variáveis (com var) e funções para o topo do seu escopo antes da execução do código. 

console.log(minhaVar); // undefined (não dá erro!)
var minhaVar = 10;
// O JS interpreta como:
// var minhaVar;
// console.log(minhaVar);
// minhaVar = 10;

// Com let e const, o hoisting existe, mas elas não são inicializadas.
// Acessá-las antes da declaração gera um Erro (Temporal Dead Zone).
// console.log(minhaLet); // ReferenceError!
let minhaLet = 20;

3.5. Coerção de Tipo (Type Coercion)

Coerção é a conversão automática de um tipo de dado para outro. Isso é a causa de muitos comportamentos estranhos no JS, especialmente com o operador ==. 

console.log(10 + "5");   // "105" (Número vira String)
console.log(10 - "5");   // 5 (String vira Número)
console.log("5" == 5);   // true (Coerção de tipo)
console.log("5" === 5);  // false (Sem coerção)
console.log(true + 1);   // 2 (true vira 1)
console.log(false + 1);  // 1 (false vira 0)
Dica: Para evitar surpresas, sempre use === e !== e faça conversões de tipo explícitas quando necessário (ex: Number("5") ou String(10)).

3.6. Template Literals (ES6)

Uma forma moderna e muito mais legível de criar strings, especialmente as que contêm variáveis (interpolação).

Elas usam crases (``) em vez de aspas.

const nome = "Ana";
const idade = 28;

// Forma antiga (concatenação)
const saudacaoAntiga = "Olá, meu nome é " + nome + " e eu tenho " + idade + " anos.";

// Forma moderna (Template Literal)
const saudacaoModerna = `Olá, meu nome é ${nome} e eu tenho ${idade} anos.`;
// Também suporta múltiplas linhas:
const html = `
    <div>
        <p>${nome}</p>
    </div>
`;

4. Estruturas de Controle

4.1. Condicionais (if, else, switch)

let hora = 14;

if (hora < 12) {
    console.log("Bom dia");
} else if (hora < 18) {
    console.log("Boa tarde");
} else {
    console.log("Boa noite");
}

// Operador Ternário (um 'if' curto)
// (condição) ? (valor se true) : (valor se false)
let saudacao = (hora < 12) ? "Bom dia" : "Boa tarde/noite";

4.2. Laços de Repetição (for, while)

// Loop `for`
for (let i = 0; i < 5; i++) {
    console.log(i);
}

// Loop `while`
let i = 0;
while (i < 5) {
    console.log(i);
    i++;
}

// Loop `for...of` (ES6) - para iterar sobre valores de arrays
const frutas = ["Maçã", "Banana"];
for (const fruta of frutas) {
    console.log(fruta);
}

// Loop `for...in` - para iterar sobre chaves de objetos
const pessoa = { nome: "Carlos", idade: 30 };
for (const chave in pessoa) {
    console.log(`${chave}: ${pessoa[chave]}`);
}

5. Funções

Funções são blocos de código reutilizáveis.

5.1. Declarando Funções

Declaração de Função (Padrão)

// Sofre hoisting (pode ser chamada antes de ser declarada)
function saudar(nome) {
    return `Olá, ${nome}!`;
}
console.log(saudar("Ana"));

Expressão de Função (Anônima)

// Não sofre hoisting
const somar = function(a, b) {
    return a + b;
};
console.log(somar(5, 3));

Veja sobre Arrow Functions na seção JavaScript Moderno.

5.2. Parâmetros (Default e Rest)

Parâmetros Padrão (ES6)

Você pode definir um valor padrão para um parâmetro caso ele não seja fornecido.

function registrar(nome, plano = "BÁSICO") {
    console.log(`${nome} está no plano ${plano}.`);
}
registrar("Bia"); // "Bia está no plano BÁSICO."
registrar("Rui", "PREMIUM"); // "Rui está no plano PREMIUM."

Parâmetros Rest (ES6)

Permite que uma função aceite um número indefinido de argumentos como um array.

function somarTudo(...numeros) {
    // 'numeros' será um array, ex: [1, 2, 3, 4]
    return numeros.reduce((total, num) => total + num, 0);
}
console.log(somarTudo(1, 2, 3, 4)); // 10

5.3. O 'this' Keyword

O this é uma das partes mais confusas do JS. Seu valor é determinado por como a função é chamada (contexto de execução).

  • Em um método de objeto: this refere-se ao próprio objeto.
  • Em uma função normal (sozinha): this refere-se ao objeto global (window no navegador) ou é undefined (em 'strict mode').
  • Em um evento: this refere-se ao elemento que disparou o evento.
  • Em Arrow Functions: this é "herdado" do escopo onde a arrow function foi criada (ela não tem seu próprio this).
const pessoa = {
    nome: "Daniel",
    saudar: function() {
        // 'this' aqui é o objeto 'pessoa'
        console.log(`Olá, eu sou ${this.nome}`);
    },
    saudarComArrow: () => {
        // 'this' aqui é 'window' (ou o escopo global)
        // pois a arrow function herdou o 'this' do escopo global
        console.log(`Meu nome é ${this.nome}`); // Provavelmente dará undefined
    }
};

pessoa.saudar(); // "Olá, eu sou Daniel"
pessoa.saudarComArrow(); // "Meu nome é undefined"

6. Estruturas de Dados: Objetos e Arrays

6.1. Objetos

Coleções de pares chave-valor. Usado para agrupar dados relacionados.

const pessoa = {
    nome: "Carlos",
    idade: 42,
    ativo: true,
    endereco: { // Objetos podem ser aninhados
        cidade: "São Paulo",
        estado: "SP"
    },
    // Um método (função dentro de um objeto)
    nomeCompleto: function() {
        return this.nome + " (idade: " + this.idade + ")";
    }
};
// Acessando propriedades
console.log(pessoa.nome); // "Carlos"
console.log(pessoa["idade"]); // 42 (útil para chaves dinâmicas)
console.log(pessoa.endereco.cidade); // "São Paulo"
console.log(pessoa.nomeCompleto());

6.2. Arrays (Matrizes)

Listas ordenadas de valores. Os índices começam em 0.

const frutas = ["Maçã", "Banana", "Laranja"];

console.log(frutas[0]); // "Maçã"
frutas.push("Uva"); // Adiciona no final
frutas.pop(); // Remove do final
frutas.unshift("Morango"); // Adiciona no início
frutas.shift(); // Remove do início
console.log(frutas.length); // 3

6.3. Métodos de Array (map, filter, reduce)

Esses são métodos (High-Order Functions) essenciais no JS moderno. Eles não modificam o array original (imutalidade) e retornam um novo array.

.map()

Transforma cada item de um array e retorna um novo array com os itens transformados.

const numeros = [1, 2, 3, 4];
const dobrados = numeros.map(n => n * 2);
// dobrados é [2, 4, 6, 8]
// numeros permanece [1, 2, 3, 4]

.filter()

Cria um novo array contendo apenas os itens que passam em um teste (retornam true).

const numeros = [1, 2, 3, 4, 5];
const pares = numeros.filter(n => n % 2 === 0);
// pares é [2, 4]

.reduce()

Reduz o array a um único valor (ex: uma soma, um objeto, etc.).

const numeros = [1, 2, 3, 4, 5];
// (acumulador, valorAtual) => ... , valorInicialDoAcumulador
const soma = numeros.reduce((total, n) => total + n, 0);
// soma é 15

6.4. Desestruturação (Destructuring) e Spread (ES6)

Desestruturação de Objeto

Uma forma fácil de extrair propriedades de um objeto para variáveis.

const usuario = {
    id: 1,
    nome: "Helena",
    email: "helena@email.com"
};
// Em vez de:
// const nome = usuario.nome;
// const email = usuario.email;

// Faça isso:
const { nome, email } = usuario;
console.log(nome); // "Helena"
console.log(email); // "helena@email.com"

// Também funciona em parâmetros de função
function saudacao({ nome }) {
    console.log(`Bem-vinda, ${nome}!`);
}
saudacao(usuario); // "Bem-vinda, Helena!"

Spread Operator (...)

Usado para "espalhar" os elementos de um array ou as propriedades de um objeto.

// Em Arrays (copiar, concatenar)
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const arrCopia = [...arr1]; // [1, 2, 3] (cópia)
const arrCombinado = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6]

// Em Objetos (copiar, mesclar)
const obj1 = { a: 1, b: 2 };
const objCopia = { ...obj1 }; // { a: 1, b: 2 }
const objMesclado = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

7. O DOM (Document Object Model)

O DOM é a representação em objeto do seu HTML. O JavaScript usa o DOM para "enxergar" e "modificar" o HTML e o CSS.

7.1. Selecionando Elementos

Os métodos preferidos hoje usam seletores CSS.

// Seleciona o *primeiro* elemento que bate com o seletor
const titulo = document.querySelector("#meu-titulo");
const primeiroBotao = document.querySelector(".btn-comprar");

// Seleciona *todos* os elementos (retorna uma NodeList)
const todosOsLinks = document.querySelectorAll("a");

7.2. Manipulando Elementos

Uma vez selecionado, você pode mudar tudo.

const titulo = document.querySelector("#titulo-principal");

// Mudar o texto (seguro)
titulo.textContent = "Bem-vindo ao JavaScript!";

// Mudar o HTML interno (cuidado com segurança)
titulo.innerHTML = "Bem-vindo ao <i>JavaScript</i>!";

// Mudar o estilo (CSS)
titulo.style.color = "blue";
titulo.style.backgroundColor = "#f0f0f0"; // camelCase para propriedades com hífen

// Adicionar/Remover classes CSS
titulo.classList.add("destaque");
titulo.classList.remove("antiga-classe");
titulo.classList.toggle("ativo"); // Adiciona se não tem, remove se tem

// Mudar atributos
const link = document.querySelector("a");
link.setAttribute("href", "https://www.google.com");
link.setAttribute("target", "_blank");

// Criar e adicionar elementos
const novoParagrafo = document.createElement("p");
novoParagrafo.textContent = "Este parágrafo foi criado com JS.";
document.body.appendChild(novoParagrafo); // Adiciona no final do

8. Eventos

Eventos são ações que acontecem na página (clique, tecla pressionada, envio de formulário).

8.1. addEventListener

A forma moderna de "ouvir" eventos.

<!-- No seu HTML -->
<button id="meu-botao">Clique Aqui!</button>
<input id="meu-input" type="text">

<!-- No seu JavaScript -->
const botao = document.querySelector("#meu-botao");
const input = document.querySelector("#meu-input");

// Evento de clique
// A função (callback) recebe um objeto 'event' com dados do evento
botao.addEventListener("click", function(event) {
    console.log("O botão foi clicado!");
    console.log(event); // Mostra o objeto do evento
});

// Evento de digitação
input.addEventListener("input", function(event) {
    // event.target é o elemento que disparou o evento (o input)
    console.log("Valor atual:", event.target.value);
});

8.2. Propagação de Eventos e preventDefault

Propagação (Bubbling)

Por padrão, quando um evento ocorre em um elemento, ele "borbulha" (bubbling) para seus elementos pais. 

// Se você clicar no <span>, os dois logs aparecerão:
// 1. "Clique no Span"
// 2. "Clique na Div" (evento borbulhou)
divPai.addEventListener("click", () => console.log("Clique na Div"));
spanFilho.addEventListener("click", () => console.log("Clique no Span"));

event.preventDefault()

Previne o comportamento padrão de um elemento. Essencial para formulários e links.

const form = document.querySelector("#meu-form");
form.addEventListener("submit", function(event) {
    // Impede que o formulário recarregue a página
    event.preventDefault(); 
    
    // Agora podemos validar os dados e enviar com JS (ex: fetch)
    console.log("Formulário enviado via JS!");
});

9. JavaScript Moderno (ES6+)

9.1. Arrow Functions (Funções de Seta)

Uma sintaxe mais curta e limpa para escrever funções, com a principal diferença de não possuir seu próprio this (ela herda o this do escopo pai).

// Função normal
const somar = function(a, b) { return a + b; };

// Arrow Function (equivalente)
const somarArrow = (a, b) => { return a + b; };

// Se tiver apenas uma linha de retorno, pode ser mais curta
const somarCurto = (a, b) => a + b;

// Se tiver apenas um parâmetro, parênteses são opcionais
const dobrar = n => n * 2;

9.2. Promises e Async/Await

JavaScript é "assíncrono" (não bloqueante). Promises são objetos que representam a eventual conclusão (ou falha) de uma operação assíncrona.

async/await (ES2017) é uma sintaxe moderna para lidar com Promises de forma mais limpa, fazendo o código assíncrono parecer síncrono. 

// "async" na função diz que ela pode usar "await"
async function buscarDadosDaAPI() {
    try {
        console.log("Buscando dados...");
        // "await" pausa a execução da função até que a Promise (fetch) seja resolvida
        const resposta = await fetch("https://api.github.com/users/google");
        
        // Esta linha só executa após o 'await' acima
        const dados = await resposta.json(); 

        console.log("Dados recebidos:", dados.name);
    } catch (erro) {
        // 'catch' pega qualquer erro que ocorra no 'try'
        console.error("Falha ao buscar dados:", erro);
    }
}

buscarDadosDaAPI();
console.log("Este log aparece ANTES dos dados serem recebidos!");

9.3. Classes (ES6)

Uma sintaxe mais limpa (syntactic sugar) para criar objetos e lidar com herança (Programação Orientada a Objetos). Por baixo dos panos, o JS continua usando protótipos (prototypal inheritance).

class Pessoa {
    // O construtor é chamado quando usamos 'new'
    constructor(nome, idade) {
        this.nome = nome;
        this.idade = idade;
    }

    // Um método da classe
    apresentar() {
        console.log(`Olá, meu nome é ${this.nome} e tenho ${this.idade} anos.`);
    }
}

// Herança
class Estudante extends Pessoa {
    constructor(nome, idade, curso) {
        super(nome, idade); // Chama o construtor da classe 'Pai' (Pessoa)
        this.curso = curso;
    }

    estudar() {
        console.log(`${this.nome} está estudando ${this.curso}.`);
    }
}

const p1 = new Pessoa("Maria", 30);
p1.apresentar(); // "Olá, meu nome é Maria e tenho 30 anos."

const e1 = new Estudante("João", 22, "Ciência da Computação");
e1.apresentar(); // "Olá, meu nome é João e tenho 22 anos."
e1.estudar(); // "João está estudando Ciência da Computação."

9.4. Módulos (import/export)

Permitem dividir seu código em múltiplos arquivos para melhor organização. Esta é a base de qualquer aplicação moderna.

Nota: Para import/export funcionar no navegador, seu script HTML principal deve ter o atributo type="module".
<script src="main.js" type="module"></script>

utils.js (Exportando)

// Exportação nomeada (named export)
export const PI = 3.14;

export function somar(a, b) {
    return a + b;
}

// Exportação padrão (default export) - só pode ter uma por arquivo
export default function subtrair(a, b) {
    return a - b;
}

main.js (Importando)

// Importando exports nomeados (precisam de chaves {})
import { PI, somar } from './utils.js';

// Importando o export default (pode ter qualquer nome)
import minhaSubtracao from './utils.js';

console.log(PI); // 3.14
console.log(somar(10, 5)); // 15
console.log(minhaSubtracao(10, 5)); // 5

10. Manipulação de Erros (try...catch)

Permite que você "tente" (try) executar um bloco de código que pode falhar. Se falhar, o bloco catch é executado, impedindo que o seu script inteiro "quebre". 

try {
    // Tenta executar este código
    let dados = JSON.parse("{ 'json_invalido': true }"); // Isso vai falhar
    console.log(dados);
} catch (erro) {
    // Se o 'try' falhar, o 'catch' é executado
    console.error("Ocorreu um erro ao processar os dados!");
    console.error("Detalhe do erro:", erro.message);
} finally {
    // O 'finally' executa sempre, independente de erro ou sucesso
    console.log("Execução finalizada.");
}

11. JSON (JavaScript Object Notation)

JSON é um formato de texto para troca de dados, muito usado em APIs. Embora pareça um objeto JavaScript, é uma String.

Diferenças: Em JSON, as chaves (keys) devem estar entre aspas duplas. Não pode haver funções, comentários ou vírgulas no final.

JSON.parse() (String → Objeto JS)

Converte uma string JSON (recebida de uma API, por exemplo) em um objeto JavaScript utilizável.

const jsonString = '{ "nome": "Bruno", "id": 123 }';
const objetoJS = JSON.parse(jsonString);

console.log(objetoJS.nome); // "Bruno"

JSON.stringify() (Objeto JS → String)

Converte um objeto JavaScript em uma string JSON (para enviar para uma API, por exemplo).

const objetoJS = { nome: "Ana", ativa: true };
const jsonString = JSON.stringify(objetoJS);

console.log(jsonString); // "{ \"nome\": \"Ana\", \"ativa\": true }"

12. Web Storage (localStorage)

Permite salvar dados simples (strings) no navegador do usuário de forma persistente.

  • localStorage: Armazena dados sem data de expiração.
  • sessionStorage: Armazena dados apenas para a sessão atual (dados são perdidos quando a aba é fechada).
// 1. Salvar dados
localStorage.setItem("tema", "dark");
localStorage.setItem("usuario", "Marcos");

// 2. Ler dados
const temaSalvo = localStorage.getItem("tema");
console.log(temaSalvo); // "dark"

// 3. Remover um item
localStorage.removeItem("usuario");

// 4. Limpar tudo
// localStorage.clear();

// Para salvar objetos, use JSON
const config = { volume: 80, legendas: true };
localStorage.setItem("configuracoes", JSON.stringify(config));

const configSalva = JSON.parse(localStorage.getItem("configuracoes"));
console.log(configSalva.volume); // 80
