Notas de Aula: Laboratório de Programação (JavaScript)

Este é um guia de consulta rápida baseado na estrutura do seu PDF "Notas de Aula"[cite: 1187, 1189, 1190]. Todos os conceitos de Python foram traduzidos para seus equivalentes em JavaScript, a linguagem da web.

1. Revisão de Algoritmos

1.1 Saída de Dados (Impressão) [cite: 1204]

Tradução: print() (Python) → console.log() (JavaScript)

Em JavaScript, a "impressão" principal para depuração é feita no console do navegador. Para exibir na tela, usamos a Manipulação do DOM.

// Python: print("Hello, World!") [cite: 1205]
console.log("Hello, World!");

// Python: print("Olá,", nome, sobrenome) [cite: 1227]
let nome = "Ana";
let sobrenome = "Silva";
console.log("Olá,", nome, sobrenome);

// Forma moderna (Template Literals):
console.log(`Olá, ${nome} ${sobrenome}`);

1.2 Variáveis [cite: 1208]

Tradução: Python (tipagem dinâmica) → JavaScript (let, const, var)

Em JS moderno, usamos let para variáveis que podem mudar de valor e const para constantes (que não podem mudar).

// Python: nome = "Ana" [cite: 1215]
let nomeCompleto = "Ana Silva";
let idade = 30;
const PI = 3.14159;

idade = 31; // Válido
// PI = 3.14; // Inválido! (Geraria um erro)

1.3 Operadores Matemáticos [cite: 1230]

Tradução: Os operadores são quase idênticos. A principal diferença é a divisão.

  • Python / (divisão real) e // (divisão inteira)[cite: 1233].
  • JavaScript / (é sempre divisão real) e Math.floor() para a parte inteira.
let x = 10;
let y = 3;

console.log(x + y); // 13
console.log(x - y); // 7
console.log(x * y); // 30
console.log(x / y); // 3.333333... [cite: 1255] (Em JS, / é sempre real)
console.log(x % y); // 1 (Resto da divisão) [cite: 1233]
console.log(x ** y); // 1000 (Exponencial) [cite: 1233]

// Equivalente ao // (divisão inteira) do Python [cite: 1233]
console.log(Math.floor(x / y)); // 3

1.4 Entrada de Dados (Leitura) [cite: 1263]

Tradução: input() (Python) → prompt() (JavaScript)

prompt() é um alerta que captura texto. Assim como no Python[cite: 1264], ele sempre retorna uma string, exigindo conversão para números.

Nota: prompt() não é muito usado. Em sites reais, usamos formulários HTML (<input>) para pegar dados.

// Python: nome = input("Digite o nome: ") [cite: 1267]
let nomeUsuario = prompt("Digite o nome: ");
console.log("Nome: ", nomeUsuario);

// Python: a = int(input("Digite o valor de a: ")) [cite: 1277]
let a = Number(prompt("Digite o valor de a: "));
let b = Number(prompt("Digite o valor de b: "));
console.log(a + b); // Soma matemática [cite: 1283]

1.5 Estrutura Condicional (if...else) [cite: 1317]

Tradução: if...elif...else: (Python) → if (...) { ... } else if (...) { ... } else { ... } (JavaScript)

A lógica é idêntica [cite: 1320-1322]. A sintaxe JS requer parênteses () na condição e chaves {} para o bloco de código.

// Python: (Código 1.12 do PDF) [cite: 1334]
// num = int(input("Digite um número: "))
// if num > 0:
//   print("O número é positivo.") [cite: 1336]
// elif num < 0:
//   print("O número é negativo.") [cite: 1338]
// else:
//   print("O número é zero.") [cite: 1340]

// JavaScript:
let num = Number(prompt("Digite um número:"));

if (num > 0) {
  console.log("O número é positivo.");
} else if (num < 0) {
  console.log("O número é negativo.");
} else {
  console.log("O número é zero.");
}

// Operadores Lógicos (PDF 1.17) [cite: 1394]
// Python: (prof=="professor" or prof=="estudante") and est=="Tocantins"
// JS:
if ((prof === "professor" || prof === "estudante") && est === "Tocantins") {
  console.log("Ganhará crédito!"); [cite: 1395]
}

1.6 Estrutura de Repetição (while) [cite: 1417]

Tradução: while condicao: (Python) → while (condicao) { ... } (JavaScript)

A sintaxe é muito similar, usando () e {}.

// Python: (Código 1.18 do PDF) [cite: 1423]
// cont = 0
// while cont < 6: [cite: 1425]
//   print(cont)
//   cont = cont + 1

// JavaScript:
let cont = 0;
while (cont < 6) {
  console.log(cont);
  cont++; // (cont++ é um atalho para cont = cont + 1)
}

1.7 Manipulação de Arrays (Listas)

Tradução: Listas (Python) → Arrays (JavaScript)

São funcionalmente idênticos. Onde o Python usa .append(), o JavaScript usa .push().

// Python: lista = []
const lista = [];

// Python: lista.append(10)
lista.push(10);
lista.push(20);
lista.push(30);

console.log(lista); // [10, 20, 30]

// Acessando um elemento
console.log(lista[0]); // 10

// Alterando um elemento
lista[1] = 25;
console.log(lista); // [10, 25, 30]

// Python: len(lista)
console.log(lista.length); // 3

2. Estrutura de Repetição (for)

Tradução: O for do JavaScript tem duas formas principais que equivalem ao for do Python.

  • Python for item in lista: (percorrer itens) → JS for...of
  • Python for i in range(5): (percorrer índices) → JS for (let i=0; ...)

Exemplo 1: for...of (Para percorrer itens)

// Python:
// frutas = ["Maçã", "Banana", "Uva"]
// for fruta in frutas:
//   print(fruta)

// JavaScript:
const frutas = ["Maçã", "Banana", "Uva"];
for (const fruta of frutas) {
  console.log(fruta);
}
// Saída:
// Maçã
// Banana
// Uva

Exemplo 2: for clássico (Para contar ou usar índices)

// Python:
// for i in range(5): # (0, 1, 2, 3, 4)
//   print(i)

// JavaScript:
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// Saída:
// 0
// 1
// 3
// 4

3. Manipulação de Matrizes (Arrays Aninhados)

Tradução: Listas de listas (Python) → Arrays de arrays (JavaScript)

A sintaxe e a lógica são idênticas às do Python.

// Python: matriz = [ [1, 2, 3], [4, 5, 6], [7, 8, 9] ]
const matriz = [
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9]
];

// Acessando o elemento da linha 1, coluna 2 (número 6)
// Python: print(matriz[1][2])
console.log(matriz[1][2]); // 6

// Percorrendo a matriz (Python: for i in range(len(matriz))...)
for (let i = 0; i < matriz.length; i++) { // Itera sobre as linhas
  for (let j = 0; j < matriz[i].length; j++) { // Itera sobre as colunas
    console.log(`Linha ${i}, Coluna ${j}: ${matriz[i][j]}`);
  }
}

4. Manipulação de Strings [cite: 983, 1001]

Tradução: Os métodos são muito parecidos, mas com nomes diferentes.

  • Python .upper() [cite: 983, 1001] → JS .toUpperCase()
  • Python .lower() [cite: 983, 1001] → JS .toLowerCase()
  • Python .strip() → JS .trim()
  • Python "fatiamento" s[0:5] → JS .slice(0, 5)
let frase = "  Laboratório de Programação  ";

console.log(frase.toUpperCase()); // "  LABORATÓRIO DE PROGRAMAÇÃO  "
console.log(frase.toLowerCase()); // "  laboratório de programação  "
console.log(frase.trim()); // "Laboratório de Programação"

// Fatiamento (Slice)
console.log(frase.trim().slice(0, 11)); // "Laboratório"

5. Funções (Functions)

Tradução: def nome_funcao(arg): (Python) → function nomeFuncao(arg) { ... } (JavaScript)

Novamente, a sintaxe JS usa function, () e {}.

// Python: (Exercício Proposto 7.8.1) [cite: 1047, 1059]
// def media(lista_de_notas):
//   soma = sum(lista_de_notas)
//   return soma / len(lista_de_notas)

// JavaScript (Forma 1: Declaração de Função)
function media(listaDeNotas) {
  let soma = 0;
  for (const nota of listaDeNotas) {
    soma = soma + nota; // ou soma += nota
  }
  return soma / listaDeNotas.length;
}

// JavaScript (Forma 2: Arrow Function)
const mediaModerna = (listaDeNotas) => {
  const soma = listaDeNotas.reduce((total, nota) => total + nota, 0);
  return soma / listaDeNotas.length;
};

// Chamando a função
const notas = [10, 8, 7, 9];
console.log(media(notas)); // 8.5
console.log(mediaModerna(notas)); // 8.5

6. Objetos (Dicionários)

Tradução: Dicionários (Python) → Objetos (JavaScript)

Este é o equivalente mais direto. Ambos são coleções de pares chave-valor. A sintaxe de acesso e modificação é muito similar.

// Python: aluno = { "nome": "Ana", "matricula": "ES256" }
const aluno = {
  nome: "Ana Maria",
  matricula: "ES256",
  notas: [9.0, 5.6, 8.5]
};

// Acessando valores (Python: aluno["nome"] ou aluno.get("nome"))
console.log(aluno.nome); // "Ana Maria"
console.log(aluno.notas[1]); // 5.6

// Alterando valores (Python: aluno["nome"] = "Ana Silva")
aluno.nome = "Ana Silva";
console.log(aluno.nome); // "Ana Silva"

// Adicionando nova chave
aluno.curso = "Engenharia de Software";

Exercício 1: Média de Idades (Baseado no Código 1.21 [cite: 914, 924])

Objetivo: Ler a idade de várias pessoas até que a idade 0 seja digitada. Calcular e mostrar a média das idades. 

// Versão interativa do Código 1.21 do PDF [cite: 914, 924]
function calcularMediaIdades() {
  let somaIdades = 0;
  let qtPessoas = 0;
  let idade = -1; // Inicia com valor diferente de 0

  // Python: while idade > 0: [cite: 924]
  while (idade !== 0) {
    idade = Number(prompt("Digite a idade (ou 0 para parar):"));

    if (idade > 0) {
      somaIdades = somaIdades + idade; // Python: soma_idades = soma_idades + idade [cite: 924]
      qtPessoas = qtPessoas + 1; // Python: qt_pessoas = qt_pessoas + 1 [cite: 924]
    }
  }

  // Exibe o resultado
  const output = document.querySelector('#output-ex1');
  if (qtPessoas > 0) {
    const media = somaIdades / qtPessoas;
    output.textContent = `Total de pessoas: ${qtPessoas}\nMédia das idades: ${media.toFixed(2)}`;
  } else {
    output.textContent = "Nenhuma idade foi digitada.";
  }
}

Exercício 2: Análise de Lista (Baseado no Exercício 3.2 [cite: 959, 971])

Objetivo: Receber uma lista de números e informar: a) a lista completa, b) a quantidade de negativos, c) a soma dos positivos. 

// Versão do Exercício Resolvido 3.2 do PDF [cite: 959, 971]
function analisarLista() {
  const numeros = [10, -5, 20, 3.5, -15, 0, 7]; // Lista de exemplo
  
  let qtNegativos = 0;
  let somaPositivos = 0;
  
  // Python: for num in lista:
  for (const num of numeros) {
    if (num < 0) {
      qtNegativos++; // b) informe a quantidade de elementos negativos [cite: 959]
    }
    if (num > 0) {
      somaPositivos += num; // c) informe a soma dos elementos positivos [cite: 959]
    }
  }
  
  // Exibe o resultado
  const output = document.querySelector('#output-ex2');
  output.textContent = `Lista Completa: [${numeros.join(', ')}]\n`; // a) imprima a lista [cite: 959]
  output.textContent += `Quantidade de Negativos: ${qtNegativos}\n`;
  output.textContent += `Soma dos Positivos: ${somaPositivos}`;
}

Exercício 3: Cadastro de Cidades (Baseado no Exercício Proposto 3 da página 91)

Objetivo: Implementar funções para gerenciar uma lista de cidades do Tocantins (imprimir, verificar data, adicionar).

Dados: [ ["Paraiso", 1963], ["Gurupi", 1958], ["Palmas", 1989], ["Miranorte", 1964] ]


// --- Banco de Dados (Global para o script) ---
// Python: cidades=[ ["Paraiso", 1963 ], ... ]
const cidades = [
  ["Paraiso", 1963],
  ["Gurupi", 1958],
  ["Palmas", 1989],
  ["Miranorte", 1964]
];

const outputEx3 = document.querySelector('#output-ex3');

// --- a) Função imprimir_cidade () ---
function imprimirCidades() {
  outputEx3.textContent = "--- Lista de Cidades ---\n";
  for (const cidade of cidades) {
    // Python: print(f"Cidade: {cidade[0]}; Ano: {cidade[1]}")
    outputEx3.textContent += `Cidade: ${cidade[0]}; Ano de criação: ${cidade[1]}\n`;
  }
}

// --- b) Função verificar_data () ---
function verificarData(anoReferencia) {
  outputEx3.textContent = `--- Verificando contra o ano ${anoReferencia} ---\n`;
  for (const cidade of cidades) {
    if (cidade[1] < anoReferencia) {
      outputEx3.textContent += `A cidade de ${cidade[0]} foi criada ANTES de ${anoReferencia}\n`;
    } else if (cidade[1] > anoReferencia) {
      outputEx3.textContent += `A cidade de ${cidade[0]} foi criada DEPOIS de ${anoReferencia}\n`;
    } else {
      outputEx3.textContent += `A cidade de ${cidade[0]} foi criada EXATAMENTE em ${anoReferencia}\n`;
    }
  }
}

// --- c) Função acrescentar_cidade() ---
function acrescentarCidade(nome, ano) {
  if (!nome || !ano) {
    outputEx3.textContent = "ERRO: Nome e Ano são obrigatórios.";
    return;
  }
  // Python: lista.append([nome, ano])
  cidades.push([nome, ano]);
  outputEx3.textContent = `Cidade "${nome}" adicionada com sucesso!\n`;
  imprimirCidades(); // Mostra a lista atualizada
}
