Guia de Consulta: Eventos e Animações

Parte 1: Eventos (JavaScript)

1. O que é um Evento?

Detalhe: Um Evento é um "sinal" que o navegador emite para o seu código JavaScript, informando que algo aconteceu na página.

O JavaScript não fica "olhando" para o botão o tempo todo. Em vez disso, ele "assina" uma lista e pede ao navegador: "Me avise quando este botão for clicado".

  • O Evento é o sinal (ex: 'click', 'scroll', 'keydown').
  • O "Ouvinte" (Listener) é a função que fica esperando esse sinal.
  • O "Callback" é a função que o JavaScript executa *depois* que o sinal é recebido.

2. O Ouvinte: addEventListener

Detalhe: Este é o método moderno e preferido para "ouvir" eventos. A sintaxe sempre tem 3 partes:

  1. O Elemento Alvo (o botão, o input, a página inteira).
  2. O Nome do Evento (uma string, ex: 'click').
  3. A Função de Callback (o que fazer quando o evento acontecer).
// 1. Seleciona o elemento
const meuBotao = document.querySelector('#btn-exemplo');

// 2. Adiciona o "ouvinte"
meuBotao.addEventListener('click', function() {
  // 3. O código aqui dentro (o callback) só roda QUANDO o botão é clicado.
  console.log('Botão foi clicado!');
});

3. Eventos Comuns (Exemplos Interativos)

Evento: click

...

// HTML: <button id="ex-btn-click">...</button>
const btnClick = document.querySelector('#ex-btn-click');
const resClick = document.querySelector('#ex-resultado-click');

btnClick.addEventListener('click', function() {
  resClick.textContent = 'Você clicou! ' + new Date().toLocaleTimeString();
});

Eventos: mouseover e mouseout

Passe o mouse aqui por cima 
const boxMouse = document.querySelector('#ex-box-mouse');

// Quando o mouse ENTRA no elemento
boxMouse.addEventListener('mouseover', function() {
  boxMouse.textContent = 'Mouse ENTROU!';
  boxMouse.style.backgroundColor = '#d4edda'; // Verde
});

// Quando o mouse SAI do elemento
boxMouse.addEventListener('mouseout', function() {
  boxMouse.textContent = 'Mouse SAIU!';
  boxMouse.style.backgroundColor = '#f8d7da'; // Vermelho
});

Evento: input

Você digitou: 

// Usado para <input>, <textarea>, <select>. Dispara a CADA letra digitada.
const inputTexto = document.querySelector('#ex-input-texto');
const resInput = document.querySelector('#ex-resultado-input');

inputTexto.addEventListener('input', function(event) {
  // O 'event.target.value' contém o texto atual do input
  resInput.textContent = event.target.value;
});

Evento: keydown

Pressione qualquer tecla...

// Para "ouvir" o teclado, geralmente adicionamos o listener no 'document' (a página inteira)
const resKey = document.querySelector('#ex-resultado-key');

document.addEventListener('keydown', function(event) {
  // O 'event.key' contém o nome da tecla pressionada
  resKey.textContent = `Você pressionou: ${event.key}`;
});

4. O Objeto 'event'

Detalhe: A função de callback (o 3º parâmetro do addEventListener) pode, opcionalmente, receber um argumento. Por convenção, chamamos esse argumento de event ou apenas e.

Este objeto event é crucial, pois contém todos os detalhes sobre o que acabou de acontecer.

A propriedade mais importante é event.target.

O que é event.target?

event.target é o elemento DOM exato que originou o evento.

Isso é útil quando você tem vários elementos filhos, mas colocou o "ouvinte" no elemento pai (uma técnica chamada "Delegação de Eventos").

  • Item 1
  • Item 2
  • Item 3
// 1. Ouvinte é colocado na <ul> (o PAI)
const lista = document.querySelector('#lista-delegacao');

lista.addEventListener('click', function(event) {
  // event.target será o <li> específico que foi clicado!
  console.log('Você clicou no: ', event.target.textContent);
  
  // Evita aplicar o estilo na <ul> caso clique entre os itens
  if (event.target.tagName === 'LI') {
    event.target.style.backgroundColor = 'yellow';
  }
});

5. event.preventDefault()

Detalhe: Alguns elementos HTML têm um "comportamento padrão" quando clicados.

  • Clicar em um link (<a>) navega para outra página.
  • Clicar em um botão type="submit" (<button>) dentro de um <form> recarrega a página.

O event.preventDefault() é um método que impede esse comportamento padrão, permitindo que o seu JavaScript assuma o controle.

...

const form = document.querySelector('#ex-form');
const resForm = document.querySelector('#ex-form-resultado');

form.addEventListener('submit', function(event) {
  // A MÁGICA ACONTECE AQUI:
  event.preventDefault(); // Impede a página de recarregar
  
  resForm.textContent = 'Formulário enviado via JS! A página não recarregou.';
});

Parte 2: Animações (CSS)

6. O que é Animação CSS?

Animação CSS é a capacidade de mudar o valor de uma propriedade CSS (como background-color, width, opacity, transform) suavemente ao longo do tempo, em vez de instantaneamente.

Existem duas maneiras de fazer isso:

  1. transition (Transição): Para animações simples, de "Estado A" para "Estado B", geralmente disparadas por uma ação (:hover) ou uma classe JS.
  2. @keyframes (Animação): Para animações complexas, com múltiplos passos (A -> B -> C), que podem rodar em loop (ex: um spinner).

7. transition (Animação A-B)

Detalhe: A transition não é uma animação em si. Ela é uma instrução que diz ao navegador: "Se alguma destas propriedades mudar, não mude instantaneamente. Mude suavemente durante X segundos."

O "disparo" (trigger) da mudança é sempre uma mudança de estado (ex: o usuário passa o mouse por cima, :hover) ou uma mudança de classe (adicionada via JavaScript).

Propriedades da Transição

  • transition-property: O que animar (ex: background-color, transform, ou all para tudo).
  • transition-duration: Quanto tempo a animação deve durar (ex: 0.5s ou 500ms).
  • transition-timing-function: A "curva de velocidade" da animação (ex: linear, ease-in, ease-in-out).
  • transition-delay: Quanto tempo esperar antes de começar (ex: 0.2s).

Exemplo 1: Disparo com CSS (:hover)

Passe o mouse sobre o quadrado:

HOVER
#box-transition {
    /* 1. Define o estado inicial */
    width: 100px;
    height: 100px;
    background-color: var(--cor-css);
    border-radius: 0%;
    transform: rotate(0deg) scale(1);
    
    /* 2. Define a instrução de transição (forma completa) */
    transition-property: all; /* Anima qualquer propriedade que mudar */
    transition-duration: 0.5s;
    transition-timing-function: ease-in-out;
    
    /* 3. Forma abreviada (shorthand) - MAIS COMUM */
    transition: all 0.5s ease-in-out;
}

/* 4. Define o estado final (o "disparo") */
#box-transition:hover {
    background-color: #c0392b; /* Muda a cor */
    transform: rotate(45deg) scale(1.2); /* Gira e aumenta */
    border-radius: 50%; /* Vira um círculo */
}

8. @keyframes e animation (Animação Multi-etapa)

Detalhe: Este é o método de animação "real". Ele funciona em duas partes:

  1. @keyframes: Você define um "roteiro" para a animação, dando um nome a ela. Você define o que acontece em 0% (ou from), 50%, 100% (ou to), etc.
  2. animation: Você "aplica" esse roteiro a um elemento, definindo a duração, quantas vezes deve repetir, etc.

Propriedades da Animação

  • animation-name: O nome do @keyframes que você criou.
  • animation-duration: A duração de UM ciclo da animação (ex: 2s).
  • animation-iteration-count: Quantas vezes repetir (ex: 3, ou infinite para sempre).
  • animation-timing-function: Curva de velocidade (linear, ease).
  • animation-direction: A direção (ex: alternate para fazer "vai-e-volta").

Exemplo 1: Loading Spinner (Loop Infinito)

/* 1. Define o "roteiro" da animação */
@keyframes spin {
    from {
        transform: rotate(0deg);
    }
    to {
        transform: rotate(360deg);
    }
}

/* 2. Aplica a animação ao elemento */
.spinner {
    width: 50px;
    height: 50px;
    border: 5px solid #f3f3f3;
    border-top: 5px solid var(--cor-css);
    border-radius: 50%;
    
    /* Forma abreviada (shorthand): nome | duração | timing | contagem */
    animation: spin 1s linear infinite;
}

Exemplo 2: Botão Pulsante (Vai-e-Volta)

/* 1. Define o "roteiro" da animação */
@keyframes pulse {
    0% { /* Estado inicial */
        transform: scale(1);
    }
    100% { /* Estado final */
        transform: scale(1.1);
    }
}

/* 2. Aplica a animação ao elemento */
.btn-pulse {
    animation-name: pulse;
    animation-duration: 1.5s;
    animation-iteration-count: infinite;
    animation-direction: alternate; /* Faz a animação ir e voltar (100% -> 0%) */
}

Parte 3: JS + CSS (A Prática Correta)

9. A Prática Correta: JS Dispara, CSS Anima

Detalhe mais importante: Evite fazer animações complexas diretamente no JavaScript (ex: elemento.style.left = ... dentro de um loop). Isso é lento e trava o navegador.

A prática profissional é "Separar as Responsabilidades":

  1. CSS: Define todos os estados da animação usando classes (ex: .slide-in, .fade-out) e as transition ou animation.
  2. JavaScript: Atua apenas como o "controlador", adicionando ou removendo essas classes no momento certo (ex: em um click).

Vantagem: O navegador otimiza as animações CSS para rodarem de forma muito mais suave (usando a placa de vídeo), e seu código JavaScript fica limpo, focado apenas na lógica (o que fazer) e não na aparência (como fazer).

10. Exemplo: Abrir/Fechar um Modal (Popup)

Aqui, o JS apenas adiciona/remove a classe .ativo. O CSS faz toda a animação de fade-in e fade-out usando opacity e visibility.

Meu Popup

Este popup foi aberto com JavaScript, mas a animação de fade-in é 100% CSS!

HTML

<!-- Botão para abrir -->
<button id="btn-abrir-modal">Abrir Popup</button>

<!-- O Modal (começa escondido) -->
<div class="modal-container" id="meu-modal">
    <div class="modal-content">
        <h3>Meu Popup</h3>
        <button id="btn-fechar-modal">Fechar</button>
    </div>
</div>

CSS

.modal-container {
    position: fixed;
    top: 0; left: 0;
    width: 100%; height: 100%;
    background-color: rgba(0,0,0,0.5);
    display: grid;
    place-items: center;
    
    /* 1. Estado Inicial: Escondido */
    visibility: hidden;
    opacity: 0;
    
    /* 2. A Transição */
    transition: opacity 0.3s ease, visibility 0.3s ease;
}

/* 3. Estado Final: Ativo (quando o JS adicionar a classe) */
.modal-container.ativo {
    visibility: visible;
    opacity: 1;
}

JavaScript

// 1. Seleciona os elementos
const btnAbrir = document.querySelector('#btn-abrir-modal');
const btnFechar = document.querySelector('#btn-fechar-modal');
const modal = document.querySelector('#meu-modal');

// 2. JS adiciona a classe 'ativo' no clique de ABRIR
btnAbrir.addEventListener('click', function() {
  modal.classList.add('ativo');
});

// 3. JS remove a classe 'ativo' no clique de FECHAR
btnFechar.addEventListener('click', function() {
  modal.classList.remove('ativo');
});
